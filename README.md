# Blog Post

## 1. What are some differences between interfaces and types in TypeScript?
TypeScript এ যখন আমরা টাইপ সেফটি নিয়ে চিন্তা করি বা নিশ্চিত করতে চাই, তখন Interface এবং  Type এ দুটোই আমাদের প্রধান হাতিয়ার। কিন্তু এখানে একটা বড় প্রশ্ন হলো আমরা কখন কোনটা ব্যবহার করব। Types নাকি interfaces? আজকে আমরা জেনে নেব এই দুটোর মধ্যেকার পার্থক্য এবং ব্যবহারের সঠিক সময়।

### Interface
ইন্টারফেস হলো একটি Contract যা অবজেক্টের Structure নির্ধারণ করে। এটি মূলত object এর আকার ডিফাইন করার জন্য ব্যবহার করা হয়। যেমন Class, object বা Function perameter এর জন্য।
```ts
interface Student {
  id: number;
  name: string;
  class: string;
  roll: number;
}
```
### Type
Type হলো একটি নামকৃত রেফারেন্স যে কোনো টাইপের জন্য। এটি শুধু অবজেক্ট নয় বরং যে কোনো টাইপকে একটি নাম দিতে পারে। এটি প্রিমিটিভ টাইপ, ইউনিয়ন, ইন্টারসেকশন, টুপল বা জটিল কম্বিনেশন ডিফাইন করতে পারে।

```ts
type Studet = {
  id: number;
  name: string;
  class: string;
  roll: number;
}
```

### Type এবং Interface এর মধ্যে পার্থক্য
TypeScript এ টাইপ এবং ইন্টারফেস প্রায় সমান কাজ করে, কিন্তু তাদের মধ্যে কিছু defference আছে। নিচে তাদের মধ্যে প্রধান পার্থক্যগুলো তুলে ধরা হলো।

| Interface | Type |
| ----- | ----- |
| `ইন্টারফেস` একাধিক ইন্টারফেস একসাথে `extend` করতে পারে, এবং ক্লাস `implements` করে ইমপ্লিমেন্ট করতে পারে | `টাইপ` সরাসরি extend করা যায় না, কিন্তু ইন্টারসেকশন & ব্যবহার করে কম্বাইন করা যায়। |
| একই নামের ইন্টারফেস একাধিকবার define করা সম্ভব এবং সেগুলো স্বয়ংক্রিয়ভাবে `মার্জ` হয়ে যায়। | `মার্জ` সাপোর্ট করে না। একই নামের টাইপ define করলে, তবে তারা ওভাররাইড হয়ে যাবে।  এরর দেয়|
| `ইন্টারফেস` শুধু অবজেক্ট-লাইক Structure এর জন্য। ইউনিয়ন বা প্রিমিটিভ টাইপ সরাসরি রিপ্রেজেন্ট করতে পারে না। | `টাইপ` ইউনিয়ন, ইন্টারসেকশন, ম্যাপড টাইপ বা কন্ডিশনাল টাইপের মতো জটিল টাইপ তৈরি করতে পারে। |

### কোনটা কখন ব্যবহার করবেন?
**ইন্টারফেস**
- অবজেক্টের স্ট্রাকচার
- ক্লাস ইমপ্লিমেন্ট
- ডিক্লারেশন মার্জিং

**টাইপ**
- ইউনিয়ন টাইপ
- টাপল
- ম্যাপড টাইপ

## 4. What is the use of enums in TypeScript? Provide an example of a numeric and string enum.
এনাম(Enum) হল TypeScript এর একটি বিশেষ ফিচার যা দিয়ে আমরা একটি গ্রুপের নির্দিষ্ট মানগুলোকে নাম দিয়ে সংরক্ষণ করতে পারি। এটি কোডকে পড়তে সহজ, বজায় রাখতে সহজ এবং টাইপ-সেফ করে।

### Enum এর সুবিধাসমূহঃ
- পঠনযোগ্যতা বৃদ্ধি
- টাইপ সেফটি
- অটোকমপ্লিট
- রক্ষণাবেক্ষণ যোগ্যতা 

### Enum দুই ধরণের
- নিউমেরিক এনাম
- স্ট্রিং এনাম 

### নিউমেরিক এনাম(Enum)
নিউমেরিক এনামের মানগুলো স্ব্যংক্রিয়ভাবে ০ থেক শুরু হয়।
```ts
enum Direction {
  Up, 
  Down,
  Left,
  Right
}
console.log(Direction.Up);
//Output: 0
```
চাইলে মানগুলো পরিবর্তন করা যায়। যদি প্রথম মান টা আমরা Up = 1 করে দেয় তাহলে পরবর্তী মানগুলো এর পর থেকে শুরু হবে।
```ts
enum Direction {
  Up = 1, 
  Down, 
  Left, 
  Right
}
console.log(Direction.Left);
//Output: 3
```

### স্ট্রিং এনাম(Enum)
স্ট্রিং এনাম(Enum) - এ প্রতিটি মান স্ট্রিং হিসাবে define করতে হয়। এই ক্ষেত্রে প্রতিটি এনাম একটি স্ট্রিং, যা ডিবাগিং এবং সিরিয়ালাইজেশন সহজ করে। 
```ts
enum UserRole {
  Admin = 'ADMIN',
  User = 'USER',
  Guest = 'GUEST'
}
console.log(UserRole.Admin);
//Output: "ADMIN"
```
